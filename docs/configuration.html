Configuration

Customize Amplify Monitor to fit your workflow.

Accessing Settings

There are several ways to access Amplify Monitor settings:

Method 1: Command Palette

  1. Press Ctrl+Shift+P (or Cmd+Shift+P on Mac)
  2. Type "Amplify Monitor: Open Settings"
  3. Press Enter

Method 2: VS Code Settings

  1. Press Ctrl+, (or Cmd+, on Mac)
  2. Search for "amplify"
  3. Modify settings as needed

Method 3: settings.json

Add settings directly to your settings.json file:

{
  "amplifyMonitor.awsProfile": "my-profile",
  "amplifyMonitor.autoRefresh": true,
  "amplifyMonitor.autoRefreshInterval": 60
}

AWS Settings

amplifyMonitor.awsProfile

Type string
Default empty (uses default profile)
Description AWS CLI profile name for authentication 
"amplifyMonitor.awsProfile": "production"

Use this when you have multiple AWS accounts and want to specify which profile to use.

amplifyMonitor.defaultRegion

Type string
Default "us-east-1"
Description Default AWS region for operations 
"amplifyMonitor.defaultRegion": "eu-west-1"
Multi-Region Support

Amplify Monitor automatically discovers apps across ALL regions, regardless of the default region setting. The default region is used for new operations and the AWS Console links.

App Defaults

amplifyMonitor.defaultAppId

Type string
Default empty
Description Default Amplify app ID for commands 
"amplifyMonitor.defaultAppId": "d1234abcdef"

Set this to skip the app selection prompt when running commands.

amplifyMonitor.defaultBranch

Type string
Default "main"
Description Default branch name for commands 
"amplifyMonitor.defaultBranch": "develop"

Auto-Refresh Settings

amplifyMonitor.autoRefresh

Type boolean
Default false
Description Enable automatic refresh of app and build data 
"amplifyMonitor.autoRefresh": true

amplifyMonitor.autoRefreshInterval

Type number
Default 60
Description Auto-refresh interval in seconds (minimum: 30) 
"amplifyMonitor.autoRefreshInterval": 120
API Rate Limits

Setting very short refresh intervals may hit AWS API rate limits. We recommend keeping the interval at 60 seconds or higher.

Notification Settings

Notification webhooks are configured through the Notifications panel, not settings.json. However, you can export/import configurations:

amplifyMonitor.notifications

Type object
Default {}
Description Webhook configurations for build notifications 
"amplifyMonitor.notifications": {
  "slack": {
    "enabled": true,
    "webhookUrl": "https://hooks.slack.com/...",
    "events": ["failed", "succeeded"]
  },
  "teams": {
    "enabled": false,
    "webhookUrl": ""
  }
}

Workspace vs User Settings

VS Code supports two levels of settings:

User Settings (Global)

Apply to all projects. Good for:

  • AWS profile (if you use the same account everywhere)
  • Auto-refresh preferences
  • Default region

Workspace Settings (Per-Project)

Apply only to the current workspace. Good for:

  • Default app ID (specific to the project)
  • Default branch (specific to the project)
  • AWS profile (if different projects use different accounts)

Creating Workspace Settings

  1. Create a .vscode folder in your project root
  2. Create a settings.json file inside it
  3. Add your project-specific settings
// .vscode/settings.json
{
  "amplifyMonitor.defaultAppId": "d1234abcdef",
  "amplifyMonitor.defaultBranch": "main",
  "amplifyMonitor.awsProfile": "client-production"
}

Example Configurations

Solo Developer

{
  "amplifyMonitor.autoRefresh": true,
  "amplifyMonitor.autoRefreshInterval": 60,
  "amplifyMonitor.defaultBranch": "main"
}

Team with Multiple Environments

{
  "amplifyMonitor.awsProfile": "team-staging",
  "amplifyMonitor.defaultRegion": "us-west-2",
  "amplifyMonitor.autoRefresh": true,
  "amplifyMonitor.autoRefreshInterval": 30
}

Consultant with Multiple Clients

// Use workspace settings for each client project
// Client A: .vscode/settings.json
{
  "amplifyMonitor.awsProfile": "client-a-prod",
  "amplifyMonitor.defaultAppId": "abc123"
}

// Client B: .vscode/settings.json
{
  "amplifyMonitor.awsProfile": "client-b-prod", 
  "amplifyMonitor.defaultAppId": "xyz789"
}
