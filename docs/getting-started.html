Getting Started

Get up and running with Amplify Monitor in under 5 minutes.

Installation

Install Amplify Monitor directly from VS Code:

Option 1: VS Code Marketplace

  1. Open VS Code
  2. Press Ctrl+Shift+X (or Cmd+Shift+X on Mac) to open Extensions
  3. Search for "Amplify Monitor"
  4. Click Install

Option 2: Command Line

code --install-extension SagaraHarasgama.amplify-monitor

Option 3: Direct Link

Install Extension

Prerequisites

Requirements
  • VS Code 1.85.0 or later
  • AWS account with Amplify apps
  • AWS credentials configured (see below)

Configuring AWS Credentials

Amplify Monitor needs AWS credentials to access your Amplify apps. Choose one of these methods:

Method 1: AWS CLI Profile (Recommended)

If you have the AWS CLI installed, configure a profile:

# Configure default profile
aws configure

# Or configure a named profile
aws configure --profile my-amplify-profile

Then set the profile in VS Code settings:

{
  "amplifyMonitor.awsProfile": "my-amplify-profile"
}

Method 2: Environment Variables

Set these environment variables before launching VS Code:

# Windows (PowerShell)
$env:AWS_ACCESS_KEY_ID = "your-access-key"
$env:AWS_SECRET_ACCESS_KEY = "your-secret-key"
$env:AWS_REGION = "us-east-1"

# macOS/Linux
export AWS_ACCESS_KEY_ID="your-access-key"
export AWS_SECRET_ACCESS_KEY="your-secret-key"
export AWS_REGION="us-east-1"

Method 3: IAM Role (EC2/ECS)

If running on AWS infrastructure, the extension automatically uses the attached IAM role.

Required Permissions

Your AWS credentials need these permissions:

  • amplify:ListApps
  • amplify:GetApp
  • amplify:ListBranches
  • amplify:ListJobs
  • amplify:GetJob
  • amplify:StartJob (for triggering builds)
  • amplify:StopJob (for cancelling builds)

First Steps

1. Open the Activity Bar

Look for the Amplify Monitor icon (⚡) in the Activity Bar on the left side of VS Code.

2. Check Connection Status

The status bar at the bottom shows your connection status:

  • ✅ Connected - AWS credentials are valid
  • ⚠️ No credentials - Configure AWS credentials
  • ❌ Error - Check your credentials/permissions

3. View Your Apps

Once connected, the Apps tree view shows all your Amplify apps across all regions.

Using the Dashboard

The Dashboard provides a visual overview of your entire Amplify portfolio.

Opening the Dashboard

  1. Press Ctrl+Shift+P (or Cmd+Shift+P on Mac)
  2. Type "Amplify Monitor: Open Dashboard"
  3. Press Enter

Dashboard Features

  • Summary Bar - Total apps, succeeded, failed, and running builds
  • App Cards - Visual grid showing all apps and their status
  • Quick Actions - Start build, stop build, view logs, open console
  • Auto-refresh - Optionally refresh data automatically

Diagnosing Failed Builds

When a build fails, Amplify Monitor can automatically diagnose the issue.

Method 1: From the Dashboard

  1. Find the failed build (shown with ❌)
  2. Click the Diagnose button
  3. View the diagnosis in the panel

Method 2: Using Command Palette

  1. Press Ctrl+Shift+P
  2. Type "Amplify Monitor: Diagnose Latest Failed Build"
  3. Select the app and branch

Understanding Diagnosis Results

The diagnosis shows:

  • Pattern - What type of error was detected
  • Root Cause - Why the error occurred
  • Suggested Fixes - Steps to resolve the issue
  • Quick Fix - One-click automated fix (if available)

Applying Quick Fixes

Many common issues can be fixed with a single click.

Available Quick Fixes

Issue Quick Fix
Lock file mismatch Switch package manager, delete conflicting lock files
Node version mismatch Create .nvmrc, update amplify.yml
npm install failed Clear cache, use legacy peer deps
Out of memory Increase Node heap size

Applying a Fix

  1. After diagnosing a build, look for the ⚡ icon next to fixable issues
  2. Click the icon to see available fixes
  3. Select a fix to apply it automatically
  4. Commit the changes and push to trigger a new build
